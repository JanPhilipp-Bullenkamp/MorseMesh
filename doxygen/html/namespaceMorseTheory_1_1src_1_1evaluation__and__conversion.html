def MorseTheory.src.evaluation_and_conversion.artifact3D_get_trafo_dict (   filename,
  scarfilename 
)

Reads the Artifact3D Softwares results and returns the reorientation they got to align the artefacts.

The reorientation returned can be used to recreate the alignment done by the Artifact3D Software as follows: Flipvector f = (f1,f2,f3); Trafomat A (3x3 Matrix); Translationvector t = (delta_x, delta_y, delta_z) A point (x,y,z) in the Artifact3D coordinate system is translated back to the original mesh coordinate system, by performing the foloowing calculations: (x_orig,y_orig,z_orig) = (f1,f2,f3) * ((x,y,z).dot(A)) - (delta_x, delta_y, delta_z) or x_orig = f * (x.dot(A)) - t So to get the Artifact3D coordinates from the original points, we have to reverse this operation: (x_art, y_art, z_art) = [(f1,f2,f3) * ((x_orig,y_orig,z_orig) + (delta_x, delta_y, delta_z))}.dot(A_inverse) or x_artifact3d = (f * (x_orig + t)).dot(A-1) (A-1 = A inverted)

Parameters
filenameAn Artifact3D .mat file. Should be called "Qins-(object_name).mat" or so.
scarfilenameAn Artifact3D .mat file containing the scar information. Should be called "ScarsQins-(object_name).mat" or so.
Returns
reorientation_trafo A dictionary containing 'Flipvector', 'Trafomatrix' and 'Translationvector' that make up the reorientation.
